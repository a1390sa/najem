# منصة إدارة المعرفة التفاعلية (Dynamic Forms Platform)

منصة ويب تفاعلية متكاملة واحترافية باللغة العربية مخصصة لإدارة وحصر وتوثيق أصول المعرفة والعمليات الحرجة والدروس المستفادة، ومتابعة المشاريع الاستراتيجية لإدارة المعرفة. تم تصميم المنصة بالكامل لدعم اتجاه **RTL** وتعمل بشكل سحابي كامل باستخدام **Supabase** كخلفية لقاعدة البيانات، ومنصة **Vercel** للاستضافة والنشر.

---

## 📁 هيكلية ملفات المشروع

يتكون المشروع من الملفات التالية في مجلد العمل:
1. **[index.html](file:///d:/doc/m9/مجلد جديد/index.html)**: واجهات المستخدم وتخطيط التبويبات والمودالات ومؤشرات الاتصال.
2. **[style.css](file:///d:/doc/m9/مجلد جديد/style.css)**: ملف التنسيقات (مظهر مؤسسي، ألوان أزرق داكن وسماوي وأبيض، خط Cairo، وتنسيقات الطباعة).
3. **[app.js](file:///d:/doc/m9/مجلد جديد/app.js)**: منطق الاتصال السحابي، الحفظ التلقائي المحلي والتحويل التلقائي (Fallback)، والتحقق والتقييم والتصدير.
4. **[config.js](file:///d:/doc/m9/مجلد جديد/config.js)**: ملف تكوين مفاتيح Supabase بشكل ثابت للمنصة.
5. **[vercel.json](file:///d:/doc/m9/مجلد جديد/vercel.json)**: إعدادات تهيئة وحماية النشر على Vercel.
6. **[database.sql](file:///d:/doc/m9/مجلد جديد/database.sql)**: كود SQL لبناء مخطط قاعدة البيانات العلائقية.

## ☁️ إعداد وتهيئة Supabase كقاعدة بيانات سحابية

لربط المنصة بقاعدة البيانات السحابية Supabase، اتبع الخطوات التالية:

### الخطوة 1: إنشاء مشروع جديد في Supabase
1. سجل الدخول إلى موقع [Supabase](https://supabase.com).
2. أنشئ مشروعاً جديداً (New Project) واختر اسماً وكلمة مرور لقاعدة البيانات، وحدد المنطقة الأقرب لمستخدميك.
3. انتظر بضع دقائق حتى يتم تجهيز قاعدة البيانات.

### الخطوة 2: إنشاء الجداول
1. من القائمة الجانبية لمشروعك في Supabase، اختر **SQL Editor**.
2. اضغط على **New Query**.
3. افتح ملف **[database.sql](file:///d:/doc/m9/مجلد جديد/database.sql)** وانسخ كود SQL بالكامل.
4. ألصق الكود في محرر SQL في Supabase واضغط على زر **Run** (أو الاختصار Ctrl+Enter).
5. تأكد من ظهور رسالة نجاح إنشاء الجداول بنجاح.

### الخطوة 3: إعداد سياسات الأمان (Row Level Security - RLS)
بشكل افتراضي، تفعل Supabase أمان الجداول مما يمنع القراءة والكتابة من المتصفح بدون مصادقة. لتسهيل بدء العمل وتجربة المنصة، يمكنك تعطيل RLS للجداول مؤقتاً لتسمح للمنصة بالاتصال المباشر عبر المفتاح العام:
- في الـ SQL Editor في Supabase، افتح سكريبت جديد وقم بتشغيل الأوامر التالية لتعطيل الحماية مؤقتاً للجداول:
```sql
ALTER TABLE submissions DISABLE ROW LEVEL SECURITY;
ALTER TABLE form1_critical_processes DISABLE ROW LEVEL SECURITY;
ALTER TABLE form2_knowledge_documentation DISABLE ROW LEVEL SECURITY;
ALTER TABLE form2_steps DISABLE ROW LEVEL SECURITY;
ALTER TABLE form3_quality_assessment DISABLE ROW LEVEL SECURITY;
ALTER TABLE form4_lessons_learned DISABLE ROW LEVEL SECURITY;
ALTER TABLE form5_strategic_project_plan DISABLE ROW LEVEL SECURITY;
```

## ⚡ ربط المنصة بقاعدة البيانات

هناك طريقتان لربط المنصة بـ Supabase:

### الطريقة الأولى: من خلال واجهة إعدادات المنصة (بدون تعديل كود)
1. افتح صفحة المنصة واذهب لتبويب **لوحة إدارة الطلبات**.
2. ستجد بطاقة باللون الأزرق الفاتح بعنوان **إعدادات الربط السحابي مع Supabase**.
3. اذهب إلى إعدادات مشروعك في Supabase (Settings -> API) وانسخ:
   - **Project URL** وضعها في خانة رابط مشروع Supabase.
   - **Anon Key** (المفتاح العام) وضعه في خانة المفتاح العام.
4. اضغط على زر **حفظ واختبار الاتصال السحابي**.
5. ستتحول الحالة في أعلى الصفحة إلى: `متصل سحابياً (Supabase) 🟢` وسيتم جلب البيانات من السحابة فوراً.

### الطريقة الثانية: كتابة المفاتيح بشكل دائم في ملف `config.js`
1. افتح ملف **[config.js](file:///d:/doc/m9/مجلد جديد/config.js)**.
2. قم بتعديل القيم وكتابة الرابط والمفتاح بين علامتي التنصيص:
```javascript
window.SUPABASE_CONFIG = {
    url: "https://your-project-id.supabase.co",
    anonKey: "your-anon-key-here"
};
```
3. احفظ الملف، وسيتصل التطبيق سحابياً تلقائياً عند تحميل الصفحة.

## 🚀 نشر واستضافة المنصة على Vercel

تعتبر منصة Vercel الأفضل لنشر تطبيقات الويب الثابتة بسرعة وأمان. لنشر هذا المشروع:

### الطريقة 1: النشر المباشر عبر Vercel CLI (سرعة وموثوقية)
1. قم بتثبيت Node.js على جهازك إذا لم يكن مثبتاً.
2. افتح نافذة الأوامر (Terminal) في مجلد المشروع ونفذ الأمر التالي لتثبيت Vercel:
   ```bash
   npm install -g vercel
   ```
3. قم بتسجيل الدخول إلى حسابك في Vercel عن طريق تشغيل:
   ```bash
   vercel login
   ```
4. ابدأ عملية النشر بالوقوف في مجلد المشروع وتشغيل:
   ```bash
   vercel
   ```
5. اتبع التعليمات في الشاشة (اضغط Enter للموافقة على الإعدادات الافتراضية). سيتم رفع الملفات فوراً وستحصل على رابط تجريبي متاح للجميع على الإنترنت.
6. للنشر الرسمي النهائي، قم بتشغيل:
   ```bash
   vercel --prod
   ```

### الطريقة 2: النشر عن طريق GitHub (التحديث التلقائي)
1. أنشئ مستودعاً جديداً (Repository) على حسابك في GitHub.
2. ارفع ملفات المشروع الستة إلى المستودع.
3. سجل الدخول إلى [Vercel](https://vercel.com) واربط حسابك بـ GitHub.
4. اضغط على **Add New -> Project** واختر المستودع الذي أنشأته.
5. اضغط على **Deploy** دون الحاجة لتغيير أي إعدادات بناء.
6. بمجرد انتهاء النشر، ستحصل على رابط رسمي على دومين `.vercel.app`. وأي تحديث ترفعه لـ GitHub سيتم نشره على Vercel تلقائياً!

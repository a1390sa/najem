    ⚙️ إعدادات الربط السحابي مع Supabase

    لربط المنصة سحابياً مع Supabase وحفظ البيانات في قاعدة بيانات حقيقية بدلاً من ذاكرة المتصفح المحلية، يرجى إدخال بيانات مشروعك هنا:

    الرقم المرجعي نوع النموذج المحتوى الأساسي تاريخ التقديم العمليات

    مخطط قاعدة البيانات العلائقية المنظم (Relational Database Schema)

    يمثل هذا المخطط الهيكل التنظيمي المقترح لتخزين وحفظ الطلبات في قواعد البيانات (مثل PostgreSQL أو MySQL أو SQLite). الجدول الرئيسي `submissions` يرتبط مع باقي الجداول عن طريق علاقة (1 to 1) باستخدام الرقم المرجعي للطلب.

    الجدول الرئيسي: submissions Primary Table
    id VARCHAR(50) (PK) الرقم المرجعي الفريد للطلب (مثال: KM-483921)
    form_type VARCHAR(20) نوع النموذج المعبأ (form1 إلى form5)
    status VARCHAR(20) حالة الطلب (draft أو submitted)
    created_at TIMESTAMP تاريخ الإنشاء الأولي للطلب
    جدول نموذج 1: form1_critical_processes FK -> submissions(id)
    id VARCHAR(50) (PK, FK) معرف الطلب المرتبط
    department_name VARCHAR(150) اسم القسم أو الإدارة
    process_name VARCHAR(200) اسم العملية الحرجة المحصورة
    knowledge_type VARCHAR(50) نوع المعرفة (implicit أو explicit)
    risk_level VARCHAR(30) مستوى المخاطرة (very_high, medium, low)
    جدول نموذج 3: form3_quality_assessment FK -> submissions(id)
    id VARCHAR(50) (PK, FK) معرف الطلب المرتبط
    asset_name VARCHAR(250) اسم الأصل المعرفي الخاضع للتقييم
    accuracy_score INT (1 to 5) تقييم الدقة
    total_score INT (5 to 25) مجموع درجات المعايير الخمسة
    accreditation_decision VARCHAR(50) القرار (approved_publish, approved_conditions, rejected)
